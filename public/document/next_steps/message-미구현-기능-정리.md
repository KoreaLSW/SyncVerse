# `/message` 미구현 기능 정리

작성일: 2026-04-06
기준 화면: `http://localhost:3000/message`

## 1) 메시지 삭제 기능

- **상태**: 미구현
- **요구사항(문서 기준)**:
    - 본인 메시지만 삭제 가능 (완료)
    - 방장은 타인 메시지도 삭제 가능 (추후 예정)
- **현재 확인 결과**:
    - 메시지 삭제 UI 버튼 없음
    - 메시지 삭제 API 엔드포인트 없음

## 2) 1:N 채팅방 생성/입장 플로우

- **상태**: 미구현 (`/message` 기준)
- **요구사항(문서 기준)**:
    - 채팅방 생성 시 이름/비밀번호/최대 인원 설정
- **현재 확인 결과**:
    - 생성 UI 없음
    - 방 생성/입장 흐름이 `/message` 화면에 연결되어 있지 않음
    - 사이드바 필터(`MAIN`, `GROUP`)는 있으나 실제 목록 데이터는 DM 중심

## 3) 메시지 무한스크롤(과거 메시지 추가 로드)

- **상태**: 부분구현
- **현재 확인 결과**:
    - 훅 레벨에서 페이징 API/`loadMore`는 구현됨
    - 화면 스크롤과 연동한 추가 로드 트리거는 미연결

## 4) 요청 상태 `만료(EXPIRED)` 전이 로직

- **상태**: 미구현
- **현재 확인 결과**:
    - 타입/DB 스키마에는 `EXPIRED` 존재
    - 자동 만료 처리(스케줄러/배치/트리거) 로직 미확인

## 5) 메시지 알림 기능(일반 메시지)

- **상태**: 부분구현
- **현재 확인 결과**:
    - DM 요청 생성 알림 저장은 구현됨
    - 일반 채팅 메시지 도착 알림(인앱/푸시/배지) 흐름은 미구현 또는 미연결

---

## 우선순위 제안

1. 메시지 삭제 기능 (사용성/관리성 즉시 개선)
2. 무한스크롤 연결 (채팅 기본 UX 완성)
3. 만료 전이 로직 (요청 상태 정합성)
4. 일반 메시지 알림 (실시간 체감 개선)
5. 1:N 생성/입장 플로우 (기능 확장)
