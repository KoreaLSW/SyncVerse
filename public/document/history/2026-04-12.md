## GROUP 채팅방 생성 기능 구현

- 사이드바 `GROUP` 필터 상태에서 생성 버튼 노출
- 그룹 생성 모달 UI 추가
  - 채팅방 이름(필수, 50자 미만)
  - 입장 비밀번호(선택)
  - 최대 인원수(필수, 2~50)
- 생성 검증 강화
  - 필수값 누락/범위 오류 시 경고창 안내
- 생성 API 보강 (`POST /api/chat/rooms`)
  - 인증/게스트 차단
  - `GROUP` 타입 생성 검증
  - 이름/정원 유효성 검증
  - 생성자 자동 participant 등록
  - `created_by` 저장

## GROUP 방 분류 조회 기능

- 분류 기준 도입
  - `전체 / My / Friend / 그 외`
- 그룹 목록 API 추가
  - `GET /api/chat/rooms/group?scope=all|my|friend|others`
  - `friendships(status='ACCEPTED')` 기반 Friend/Others 분류
- 사이드바 하위 필터 버튼 UI/상태 연동

## GROUP 참여(입장) 흐름 구현

- 접근 상태 API 추가
  - `GET /api/chat/rooms/[id]/access`
  - 참여 여부/비밀번호 필요 여부/현재 인원/최대 인원/참여 가능 여부 반환
- 입장 API 보강
  - `POST /api/chat/rooms/[id]/join`
  - 비밀번호 검증(`WRONG_PASSWORD`)
  - 정원 초과 검증(`ROOM_FULL`)
  - 이미 참여 중일 때 idempotent 성공 처리
- 중앙 채팅 영역 참여 UI 추가
  - 미참여 그룹방에서 메시지 대신 참여 패널 노출
  - 비밀번호 입력 후 참여 가능
- 미참여 상태 메시지 API 호출 차단
  - 403 반복 로그 문제 완화

## GROUP 나가기/삭제 정책 구현

- 정책
  - 방장(생성자): 채팅방 삭제

  - 방장은 `채팅방 삭제`, 참여자는 `나가기`로 분기
  - 처리 후 목록/선택/요약 상태 정리
