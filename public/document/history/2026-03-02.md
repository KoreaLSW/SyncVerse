# 2026-03-02 작업 기록

## 알림(Notification) 기능 확장

- `notifications` 기반 API 1단계 구현
  - `GET /api/notifications` (목록 조회, limit/cursor/unreadOnly/type 지원)
  - `POST /api/notifications/read-all` (전체 읽음 처리)
  - `PATCH /api/notifications/[id]/read` (단건 읽음 처리)
- 친구 요청 생성 시 알림 row 저장 연동
  - `POST /api/friends/request`에서 `FRIEND_REQUEST` 알림 insert
  - `payload`에 `friendshipId`, `senderId`, `receiverId`, `senderNickname` 저장
- 클라이언트 알림 스토어 및 Realtime 반영
  - `stores/notificationsStore.ts` 추가
  - 초기 로드 + `notifications` 테이블 `INSERT/UPDATE/DELETE` 실시간 구독
  - `unreadCount` 계산/유지
- 알림 패널 UI 구현
  - `components/NotificationPanel.tsx` 추가
  - 미읽음 배지, 목록 표시, 개별 읽음/전체 읽음 기능
- 친구 요청 알림에서 즉시 액션 처리
  - `POST /api/notifications/[id]/act` 추가
  - 알림 패널에서 `수락/거절` 버튼 제공
  - 처리 후 `acted_at`, `read_at` 반영
- 알림 삭제 기능 추가
  - `DELETE /api/notifications/[id]` (단건 삭제)
  - `DELETE /api/notifications?readOnly=true` (읽은 알림 일괄 삭제)
  - 패널에 `삭제`, `읽은 알림 삭제` 버튼 연결
- 친구 요청 수락 알림 발송 추가
  - `POST /api/friends/accept` 성공 시 요청자에게 수락 알림 발송
  - `/api/friends/request`의 자동 수락 분기에서도 동일 알림 발송
