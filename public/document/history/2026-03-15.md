# 2026-03-15 작업 기록

## 메시지(Message) 실사용 전환

- 임시 목데이터 의존 제거
  - `ROOM_MOCK`, `MESSAGE_MOCK` 기반 초기 렌더 제거
  - 실제 API/Realtime 데이터 흐름으로 메시지 화면 동작하도록 전환
- DM 수락 후 방 노출 로직 고도화
  - `chat_requests.resolved_room_id`를 기반으로 수락된 DM 방을 사이드바에 반영
  - 새로고침 이후에도 DM 리스트 유지되도록 상태 구성

## 메시지 전송/저장/실시간 반영

- 메시지 페이지에서 로컬 임시 append 방식 제거
  - `useChat(roomId)` 기반으로 메시지 조회/전송 연결
- `POST /api/chat/messages`로 DB 저장 후 Realtime으로 상대 화면 반영
- `GET /api/chat/messages`/`POST /api/chat/messages` 보안 보강
  - 인증 체크
  - 게스트 차단
  - `chat_participants` 참가자 검증
  - 본인 `sender_id` 일치 검증

## 사이드바 최신 메시지/시간/미읽음 반영

- 방 목록 요약 API 추가
  - `GET /api/chat/rooms/summary?roomIds=...`
  - `latestMessage`, `latestAt`, `unreadCount` 계산/반환
- 읽음 처리 API 추가
  - `POST /api/chat/rooms/[id]/read`
  - `chat_participants.last_read_at` 업데이트
- 메시지 페이지에서 summary API 결과를 방 리스트에 병합
- `messages INSERT` Realtime 이벤트 수신 시
  - 최신 메시지/시간 갱신
  - 현재 보고 있지 않은 방은 `unreadCount` 증가

## 읽음(Seen) 표시 및 실시간 읽음 동기화

- 메시지 타입 확장
  - `MessageItem.isReadByPeer` 추가
- 메시지 조회 API에서 `is_read_by_peer` 계산 반환
  - 상대 참여자의 `last_read_at`과 내 메시지 `created_at` 비교
- 메시지 버블 하단에 `읽음` 표시 UI 추가
- 읽음 실시간 반영 보완
  - `chat_participants UPDATE` Realtime 구독 시 메시지 재검증
  - 상대방 읽음 갱신이 발생하면 `읽음` 상태 즉시 반영

## 읽음 처리 트리거/과호출 제어 개선

- 읽음 처리 함수 공통화 + throttle 적용 (`700ms`)
- 트리거 조건 고도화
  - 방 전환 시 즉시 읽음 처리
  - 현재 방에서 상대 메시지 신규 수신 + 탭 활성 + 하단 스크롤 상태일 때 읽음 처리
  - 탭 재활성화 + 하단 상태일 때 읽음 처리

## 스크롤 UX 개선

- 채팅 영역 높이/overflow 구조 보완으로 메시지 많을 때 내부 스크롤 정상화
- 채팅방 초기 진입/전환 시 스크롤 자동 하단 이동
- 전송 시(스크롤이 위에 있어도) 하단으로 자동 이동
- 새 메시지 수신 시 하단으로 자동 이동
- 다크 테마 분위기에 맞는 커스텀 스크롤바 스타일 적용

## 추후 보완 후보

  - 현재는 DM 중심의 `읽음` 계산
  - 그룹에서는 `전원 읽음/몇 명 읽음` 정책 정의 필요
- 메시지 사이드바 정렬 기준(최근 메시지 시각 우선) 고도화
