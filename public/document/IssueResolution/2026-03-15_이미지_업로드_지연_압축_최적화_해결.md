# 이미지 업로드 지연 이슈 해결 (2026-03-15)

## 발생 이슈

- 증상
  - 사진 전송 시 업로드 시간이 길어 사용자 체감이 느림
  - 특히 고해상도 원본 이미지에서 전송 대기 시간이 크게 증가

- 영향
  - 채팅 이미지 전송 UX 저하
  - 전송 완료까지 사용자가 기다리는 시간이 길어짐

## 원인 분석

- 이미지 파일이 업로드 전에 리사이즈/압축되지 않고 원본 그대로 전송됨
- 업로드 병목은 주로 사용자 상행 회선(업로드 대역폭)에서 발생하므로, 파일 바이트 수가 클수록 지연이 커짐

## 해결 방법

- 클라이언트 업로드 전 압축 파이프라인 추가
  - 긴 변 기준 리사이즈: `1600px`
  - 출력 포맷: `webp`
  - 품질 단계 압축: `0.82 -> 0.76 -> 0.68 -> 0.6`
  - 목표 용량: `2MB` 이하

- 안정성 처리
  - 압축 결과가 원본보다 큰 경우 원본 사용
  - `gif`는 애니메이션 손실 방지를 위해 원본 유지
  - 압축 실패 시에도 전송 자체는 가능하도록 원본으로 폴백

- UX 처리
  - 압축 중 상태(`압축 중...`)와 업로드 중 상태(`업로드 중...`)를 구분해 표시
  - 압축/업로드 중에는 카메라 버튼 및 전송 버튼 비활성화

## 수정 파일

- `syncverse/lib/message/imageCompression.ts`
- `syncverse/app/message/page.tsx`
- `syncverse/components/message/MessageChatSection.tsx`

## 기대 효과

- 업로드 바이트 감소로 전송 지연 단축
- 대용량 이미지 전송 시 체감 속도 개선
- 사용자에게 현재 처리 상태를 명확히 안내하여 UX 개선

## 검증 포인트

- 같은 이미지 업로드 시 기존 대비 전송 완료 시간이 단축되는지 확인
- 압축 중/업로드 중 상태 문구 및 버튼 비활성화 동작 확인
- `gif` 업로드 시 애니메이션이 유지되는지 확인
- 압축 실패 상황에서도 업로드가 중단되지 않고 원본 전송되는지 확인
